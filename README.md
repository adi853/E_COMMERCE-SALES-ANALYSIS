# E_COMMERCE-SALES-ANALYSIS
## E_COMMERCE SALES ANALYSIS
## Project Overview
This project focuses on analyzing an e-commerce dataset containing 11 months of sales data across 30+ product categories. Using MySQL, I extracted actionable insights regarding revenue trends, seasonality, and customer satisfaction to help drive data-driven business decisions.
***Revenue Leader:*** The ***BOOKS,SPORTS,TOYS,CLOTHING,HEALTH*** category emerged as the primary revenue driver, contributing significantly to the total turnover.
***Sales Seasonality:*** Analysis revealed a significant sales peak in Quarter 3, suggesting a strong seasonal trend toward the end of the year.
***Customer Satisfaction:*** High-rated products (Review Score > 4.5) show a strong correlation with higher sales volume, highlighting the link between quality and revenue.
## SQL Skills Showcased
***Data Aggregation:*** Utilizing SUM, AVG, and COUNT to calculate key performance indicators (KPIs).
***Advanced Grouping:*** Using GROUP BY to analyze performance across different categories.
***Arithmetic Calculations:*** Creating calculated fields for total revenue and cumulative sales across 11 months.
***Seasonality Logic:*** Segmenting data into Quarters (Q1, Q2, Q3) to identify growth patterns.
## Repository Structure
***analysis_queries.sql:*** The main script containing all SQL queries, from table setup to advanced analysis.
***dataset.csv:*** The raw sales data used for the analysis.
***Presentation_Slides.pdf:*** Professional case study slides summarizing the findings.
